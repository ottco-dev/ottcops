Overview

The Analyzer pairs Teachable Machine models with LLM outputs and returns clean JSON. The front end and backend stay classic: FastAPI + HTML/JS/CSS without extra frameworks. Every feature can run independently and is documented here.

Highlights

  • Prompt Templates & Builder: Auto-prompts, FAQ drag-and-drop tiles, and per-browser custom presets.
  • Batch & ML-only: Multi-image uploads and pure TM inference without LLMs.
  • Debug Panel: Request IDs, model versions, and timing breakdowns.
  • API Token Mode: Custom base URL + token for enterprise targets.
  • API Endpoints Explorer: Custom-styled OpenAPI playground to test every route without Swagger.
  • UI Enhancements: Stream dashboard, theme switch, JSON fullscreen, drag & drop.
  • Export: JSON download, PDF report, and share links with a viewer.
  • Video & Streams: Scheduler for snapshot/RTSP cameras with batch automation.
  • ML Models: Tutorials for Teachable Machine (easy) and Label Studio + YOLO (pro).
  • Custom Modules: How to attach your own backend modules and front-end hooks.
  • Raspberry Pi Guides: Mounting in grow tents, camera setup, edge scripting.
  • Home Automation: Examples for Home Assistant, Node-RED, and Python scripts.
  • LLM Switchboard: `/api/settings/llm` persists provider/base URL/model/system prompt for Analyzer & OTTO.

Code reference & modular tips

  • Routing: The FastAPI app is instantiated in app.py after importing shared helpers from opencore/. Each feature mounts its own endpoint module (e.g., batch, uploads, MQTT) so you can disable or extend them independently.
  • Front-end controller: static/js/app.js holds feature-specific functions such as submitAnalysis(), submitBatchAnalysis(), and refreshMqttSensors(). Each function is isolated so you can lift it into your own pages or call it conditionally.
  • Configuration store: opencore/settings_store.py persists LLM profiles, MQTT brokers, and other optional blocks; every setter has a matching getter for selective use in custom endpoints.

When adapting the stack, start by importing only the functions you need (for example, from opencore.settings_store import load_llm_profiles) and wire them into new routers or UI buttons. The existing docs detail each module so you can plug pieces in without carrying the whole app.
