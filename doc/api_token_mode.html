API-Token-Mode

Mit dem Professional Mode lässt sich jede Instanz gegen eine eigene API-Base-URL und ein Token richten.

Zurück Export

Konfiguration

  1. Auf der Analyzer-Seite das Zahnrad oben rechts anklicken.
  2. Base-URL (z. B. https://opencore.partner.tld) eintragen.
  3. API-Token eingeben. Das Feld ist als Passwortfeld ausgelegt.
  4. Toggle „API-Token-Mode aktivieren“ einschalten und speichern.

Die Daten werden ausschließlich im lokalen Browser gespeichert (localStorage.opencore_settings).

Requests

Solange der Modus aktiv ist, baut das Frontend alle Requests gegen die konfigurierte Base-URL und setzt automatisch einen Authorization: Bearer <token>-Header.

# curl
curl -X POST "https://partner/api/opencore/analyze" \
  -H "Authorization: Bearer <token>" \
  -F "prompt=Bag Appeal" \
  -F "image=@bud.jpg"

# Python
import requests
url = "https://partner/api/opencore/analyze"
headers = {"Authorization": "Bearer <token>"}
files = {"image": open("bud.jpg", "rb")}
data = {"prompt": "Trichome-Reifegrad"}
resp = requests.post(url, headers=headers, files=files, data=data)
print(resp.json())

# Node.js (fetch)
const form = new FormData();
form.append("prompt", "Pest-Detection");
form.append("image", fs.createReadStream("bud.jpg"));
const res = await fetch("https://partner/api/opencore/analyze", {
  method: "POST",
  headers: { Authorization: `Bearer ${token}` },
  body: form,
});
console.log(await res.json());

Sicherheit

  • Das Backend loggt den Token nicht.
  • Browser-Speicher kann manuell geleert werden (Button „Einstellungen zurücksetzen“ im Modal).
  • Bei deaktiviertem Modus laufen alle Requests wieder lokal gegen /analyze bzw. /api/opencore/*.
